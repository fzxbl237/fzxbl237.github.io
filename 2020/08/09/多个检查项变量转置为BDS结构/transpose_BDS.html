2. 思路分析

2.1 初始思路

刚开始遇到这种需求，第一反应的做法可能是：

data res;
retain subjid visitnum lbtestcd lborres lborresu lbclsig;
length lborres $20. lborresu $40. lbclsig $40.;
set lb;

**lbtestcd='WBC';
    lbtestcd="WBC";    
    lborres=lborres_wbc;
    lborresu=lborresu_wbc;
    lbclsig=lbclsig_wbc;
    output;
**lbtestcd="ALT";
    lbtestcd="ALT";    
    lborres=lborres_alt;
    lborresu=lborresu_alt;
    lbclsig=lbclsig_alt;
    output;
keep subjid visitnum lbtestcd lborres lborresu lbclsig;
run;

这么做的==缺点==是：

  • 实际上LB的检查项是比较多的（大概可能有40-50个），那么这么写代码就太臃肿，手也会疼；
  • 需要注意控制lborres、lborresu、lbclsig这三个新增变量的length，避免数据截断；

2.2 转置思路

遇到这种增加观测的需求，可能需要考虑的是转置，但是可能直接转置达不到要求，因为lborres、lborresu、lbclsig需要在转置后仍然需要作为三个变量存在，而非一个；

幸运的是，上述六个变量的规律性很强，他们分别是以LBORRES、LBORRESU、LBCLSIG作为前缀，所以我们需要考虑需要用多次转置来达到我的要求

  • 先将rawdata LB的变量清单储存在varlist宏变量中，后面的代码会用到这个宏变量
** 将rawdata LB的原始变量列表记录到varlist宏变量中;
proc sql noprint;
    select 
        name 
            into:varlist separated by " "
        from sashelp.vcolumn
        where libname="WORK"
            and memname="LB";
quit;
  • 对于rawdata -LB不要先处理其他变量，先转置成BDS结果再处理和筛选可能会方便一点;所以我们直接sort by 所有变量即可
**对于rawdata -LB不要先处理其他变量，先转置成BDS结果再处理和筛选可能会方便一点;
proc sort data=lb out=lb1;
    by _all_;
run;
  • 对我们需要的lborres、lborresu、lbclsig类变量转置
proc transpose data=lb1 out=lb2;
    by _all_;
    var lborres_: lborresu_: lbclsig_:;
run;
  • 截取出LBTESTCD和_VARCAT变量，其中VARCAT变量就是我们转置后的变量分类，它的取值即为[“LBORRES”、“LBORRESU”、”LBCLSIG”]，它将作为第二次转置的==ID==
data lb3;
    set lb2;
    lbtestcd=scan(_name_,-1,"_");
    _varcat=scan(_name_,1,"_");
run;
  • 第二次转置即可得到我们期望的结果;
**第二次转置即可得到我们期望的结果;
proc sort data=lb3;
    by &varlist. lbtestcd;
run;

proc transpose data=lb3 out=lb4;
    by &varlist. lbtestcd;
    var col1;
    id _varcat;
run;
  • 最后的筛选操作即可得到我们需要的转置结果
data res;
    set lb4;
    keep subjid visitnum lbtestcd lborres lborresu lbclsig;
run;

3.总结与讨论

上述的转置代码乍一看比第一种代码麻烦一些，但是上述代码的优点有：

  • 上述转置代码量不会因为检查项的增多而增多，即使成百上千个检查项，也依然只需要上述的代码量即可，当然前提是：==所有的检查项相关变量具有统一的命名规则，当然实际项目中如果遇到上述数据结构，只要是稍微正规点的EDC系统基本上会用统一的规则命名，如果检查项相关变量真的没啥规则，那可能确实没啥简单的方法==；

  • 上述代码不需要控制输出变量的长度，即不需要担心输出变量会因为长度不够而被截断的问题；

  • 上述转置代码的ASSIGN语句少，耦合性可能会好一点（就是这么不要脸）；

  • 上文的代码只是==针对问题情景我个人的一点思路==，肯定不会是最优解，如果有问题，那你TM来打我啊。
